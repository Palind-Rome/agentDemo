# `session.py`

Message 是一个数据类，包括角色、文本内容、工具调用等，对应使用中的单挑消息。

Session 也是数据类，每个 Session 包括多条 Message。其有两个函数，添加 Message 函数和获取 Message 函数，last_assistant、last_assistant_text 函数找到最后一条 assistant 消息文本。

# `tool.py`

ToolParameter 包含工具参数的 name、type、description、required。

一个 Tool 包含其 name、给 LLM 看的 description、由 ToolParameter 组成的参数列表、可执行对象 execute_fn 作为工具的执行逻辑。其有两个函数，可以执行工具返回文本结果；可以将name转换为 OpenAI function calling 这种特定的 JSON 格式，便于 LLM 阅读。

最后手动实现了几个内置工具，包括读取文件内容工具、写入/创建文件工具等。首先将它们的函数写好，其完成操作后会返回执行结果。然后，为这些工具注册：

```python
# 所有可用的工具
TOOL_REGISTRY: dict[str, Tool] = {}

def register_tool(tool: Tool):
    """注册一个工具"""
    TOOL_REGISTRY[tool.name] = tool
```

像这样传入工具的名字、描述、参数、执行函数：

```python
register_tool(Tool(
    name="read_file",
    description="读取指定文件的内容。当你需要查看代码或文件时使用。",
    parameters=[
        ToolParameter("file_path", "string", "要读取的文件路径"),
    ],
    execute_fn=_read_file,
))
```

最后用 get_tools_for_agent 返回一个 agent 可用工具列表。

# `permission.py`

每个 agent 有一套权限规则 permissions dict 作为约束，工具执行前会用 enforce_permission 函数检查 Agent 是否有权使用这个工具，返回的动作为 allow、deny、ask（询问用户能否使用）。

# `hook.py`

只要在代码关键位置预留 hook 点，外部插件就可以往插槽里注册自己的处理函数。当代码执行的 hook 点时，遍历调用函数数组中所有注册的函数。具体实现如下。

HookManager 类维护一个 dict: `{ "hook名称": [回调函数列表] }`，可以向 dict 里面注册函数。trigger 函数会触发一个 hook，逐个调用所有注册的回调函数。

例如，编写一个安全审计插件，功能是记录所有工具的调用，

```python
def create_audit_plugin():
    """
    创建一个安全审计插件
    记录所有工具的调用

    返回一个函数，接受 HookManager 并注册 hook 回调。
    对应 OpenCode 中插件的 init(hooks) 模式。
    """
    log = []

    def on_tool_before(input_data: dict, output_data: dict):
        tool_name = input_data.get("tool", "unknown")
        args = output_data.get("args", {})
        entry = f"[审计] 工具 '{tool_name}' 即将执行，参数: {args}"
        log.append(entry)
        print(f"  🔍 {entry}")

    def on_tool_after(input_data: dict, output_data: dict):
        tool_name = input_data.get("tool", "unknown")
        result = output_data.get("result", "")
        preview = result[:100] + "..." if len(result) > 100 else result
        entry = f"[审计] 工具 '{tool_name}' 执行完成，结果: {preview}"
        log.append(entry)

    def register(hooks: 'HookManager'):
        """将审计插件注册到 HookManager"""
        hooks.register("tool.execute.before", on_tool_before)
        hooks.register("tool.execute.after", on_tool_after)

    return register
```

之后只需要在 `main.py` 中写入

```python
    hooks = HookManager()
    audit_plugin = create_audit_plugin()
    audit_plugin(hooks)
    print("✅ Hook 系统已初始化（审计插件已加载）")
```

再在进入 agentic_loop 的时候传入 hooks：

```python
            result = agentic_loop(
                session=session,
                agent=current_agent,
                user_message=user_input,
                hook_manager=hooks,
                depth=0,
            )
```

在 `agentic_loop` 中，会有：

```python
            # ---- 4b. 触发 hook: tool.execute.before ----
            hook_output = {"args": args}
            hooks.trigger("tool.execute.before", {"tool": tool_name, "session_id": session.id}, hook_output)
            args = hook_output["args"]  # 插件可能修改了参数
```

以及：

```python
            # ---- 4d. 触发 hook: tool.execute.after ----
            hook_output = {"result": tool_result}
            hooks.trigger("tool.execute.after", {"tool": tool_name, "session_id": session.id}, hook_output)
            tool_result = hook_output["result"]  # 插件可能修改了结果
```

# `loop.py`

```python
核心思想：
  while True:
      1. 把历史消息 + 工具列表发送给 LLM
      2. LLM 返回响应
      3. 如果 LLM 说"我说完了"(finish_reason="stop") → break
      4. 如果 LLM 说"我要调用工具"(finish_reason="tool_calls"):
         a. 检查权限
         b. 触发 hook
         c. 执行工具
         d. 把工具结果加入消息历史
         e. continue（回到步骤 1）

  LLM 自己决定什么时候停下来。开发者不控制循环次数。
```
